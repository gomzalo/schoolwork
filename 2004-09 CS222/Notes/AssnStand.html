74.222 Introduction to Assembly Language Programming

Assignment Standards

Assignments

  • Assignments should be in a letter size folder with your name (family name underlined) and student number prominently displayed on the outside.
  • Include your Honesty Declaration.
  • Include printouts of the source code and output from your programs.

Programming Style - source file

  • Put your name, student number, the course number, instructor's name, the assignment number, and the question number in comments at the start of the program source.
  • Include a brief but precise description of the program in the comments. Include a description of the inputs expected, the processing that is done, and the output that is produced.
  • Use blank lines to separate major sections of code and to improve readability.
  • Include a brief comment before each major section of code indicating what it does.
  • Include a "register dictionary" before each major section of code that indicates what the registers are used for in that section.
  • Many assembly language instructions should contain a brief comment. The comment should be useful. For example, on the instruction "move D3,D0" the comment "moves D3 to D0" is useless, but the comment "prepare to print D3 with decout" is appropriate.
  • Use synthetic instructions in SPARC code.
  • Use "bottom testing" loops (perhaps with rare exceptions).
  • In all subroutines, include comments on the purpose, the input parameters, and the output parameters.

Programming Style - output file

  • Print your name, student number, the course number, instructor's name, the assignment number, and the question number (as a "banner") at the start the output of your program.
  • Make the output of your readable on its own (that is, include titles or labels in the output). Always prompt the user for input, almost always echo the input.
  • At the end of execution, print a short message like "End of Processing".
